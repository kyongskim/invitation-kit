# 프로젝트 기획서 — 모바일 청첩장 템플릿

> 1주차 산출물 · 의사결정 근거 문서

---

## 한 줄 소개 (Elevator Pitch)

**한국 결혼식에 최적화된 오픈소스 모바일 청첩장 템플릿. 설정 파일 하나만 수정하면 Vercel에 5분 안에 배포되는 `config-driven` 구조.**

---

## 왜 만드는가 (Problem)

1. 기존 상용 모바일 청첩장은 월 구독 또는 유효기간이 있어 시간이 지나면 사라짐
2. 디자인 자유도가 낮고 광고/워터마크가 붙는 경우가 많음
3. 개발자가 만든 오픈소스 청첩장은 대부분 **개인용 하드코딩**이라 다른 사람이 재사용하려면 소스를 직접 뜯어야 함
4. 해외 오픈소스 템플릿(wzulfikar, Weddingly-Free 등)은 **한국 특화 기능이 없음** — 카카오톡 공유, 네이버/카카오 지도, 한국 계좌번호 복사, 축의금 송금(토스/카카오페이)

## 누구를 위한 것인가 (Target Users)

- **Primary:** 주니어~시니어 프론트엔드 개발자 (본인 결혼 또는 친구 선물용)
- **Secondary:** Fork → 이미지/텍스트만 바꿔서 쓰고 싶은 비전공 개발자
- **Stretch:** 비개발자 (향후 웹 에디터 UI 제공 시)

## 해결 방법 (Solution)

- `invitation.config.ts` 한 파일만 수정하면 전체 청첩장이 바뀌는 구조
- Fork → Vercel 원클릭 배포 워크플로우 (5분 이내 완료)
- 한국 결혼식 필수 기능 기본 내장 (카카오 공유, 한국 지도 3종, 계좌 복사)
- 3~5종의 기본 테마 제공 (모던 / 클래식 / 플로럴 / 미니멀 / 빈티지)

## 이름 후보

| 이름 | 장점 | 단점 |
|------|------|------|
| **`invitation-kit`** | 명확, 검색 용이, 확장성 (생일/돌잔치 등) | 약간 일반적 |
| **`wedding-kit`** | 짧고 외우기 쉬움, Wedding Kit SDK 느낌 | 범위가 결혼에 한정 |
| **`moblie-cheongcheopjang`** | 강한 한국 정체성 | 외국인 개발자 접근 어려움, 오타 소지 |
| **`haru-invitation`** ("하루") | 감성적, 한국어이지만 발음 쉬움 | 이미 유사 이름 존재 가능 |
| **`woori-wedding`** ("우리") | 친근함, 한국 정체성 | 은행 이름 연상 |
| **`yebin`** ("예빈" = 예식의 빈 자리) | 독특, 브랜드화 가능 | 의미 전달이 약함 |

**추천:** `invitation-kit` 또는 `haru-invitation`
- `invitation-kit`: 장기적으로 청첩장 외에 돌잔치, 회갑 등 다양한 초대장 템플릿으로 확장 가능
- `haru-invitation`: 단일 프로덕트로서 브랜딩이 강함, "특별한 하루"라는 컨셉과 연결

**결정은 본인의 취향에 따라** — 이 문서의 나머지는 `invitation-kit`으로 표기.

## 기술 스택 확정

| 영역 | 선택 | 근거 |
|------|------|------|
| 프레임워크 | Next.js 15 (App Router) | 정적 생성 + 이미지 최적화, Vercel 최적 궁합 |
| 언어 | TypeScript | config 스키마 타입 힌트 제공 = 사용자 경험 ↑ |
| 스타일 | Tailwind CSS | 반응형 속도, shadcn/ui 호환 |
| 애니메이션 | Framer Motion | 스크롤 페이드인, 자연스러운 전환 |
| 백엔드 | Firebase Firestore | 무료 티어 넉넉, 방명록/RSVP에 충분 |
| 배포 | Vercel | GitHub 연동 자동배포 + 무료 SSL |
| 지도 | 네이버/카카오/구글 지도 버튼 연결 | API 비용 없이 딥링크로 해결 |
| 공유 | Kakao JavaScript SDK | 한국 결혼식 공유는 사실상 카카오톡 전용 |

**비용:** 월 ₩0 (Vercel Hobby + Firebase Spark + 카카오/네이버 무료 API)
